# 低成本 (<3000元) ADV Moto 平台启动方案

## 1. 核心思路：从“精益创业”到“社区自举”
预算有限意味着我们不能烧钱做推广，也不能购买昂贵的商业地图服务。我们的策略必须转变为：
**用内容换流量，用工具留存用户，用社群做护城河。**

## 2. 零成本技术栈选型 (The "Zero-Cost" Stack)

| 模块 | 传统高成本方案 | **我们的零成本替代方案** | 成本预估 |
| :--- | :--- | :--- | :--- |
| **前端托管** | 阿里云/AWS EC2 (¥200+/月) | **Vercel / GitHub Pages** (静态托管) | ¥0 |
| **数据库** | 阿里云 RDS (¥300+/月) | **Supabase** (Free Tier: 500MB, 足够的初期用量) | ¥0 |
| **地图服务** | Google Maps / Mapbox (付费API) | **MapLibre GL JS** + **OpenStreetMap (OSM)** | ¥0 |
| **后端逻辑** | 独立服务器 Node.js | **Supabase Edge Functions** (Serverless) | ¥0 (含在免费额度内) |
| **域名** | .com 域名 | **.top / .xyz ** (首年极低，或用 Vercel 免费二级域名测试) | < ¥50/年 |
| **AI 能力** | GPT-4 API (昂贵) | **DeepSeek API** 或 **Kimi API** (目前极低成本) | < ¥100/月 |

**初期总硬件成本预估：¥100 以内** (仅域名 + 少量 API 费用)。

## 3. 运营模式：Concierge MVP (礼宾式最小可行性产品)
不要一开始就开发全自动的 AI系统。初期用 **“人工 + 辅助工具”** 的形式服务用户。

### 阶段一：微信社群 + "人工"智能 (前 1-3 个月)
*   **产品形态**：一个微信群 + 一个 Notion/飞书 公开文档页面。
*   **服务流程**：
    1.  用户在通过微信发需求：“下周想去安捷伦玩泥巴，中等难度。”
    2.  你（作为管理员）使用后台的 **AI 辅助工具**（我们开发的简易版）快速生成路线。
    3.  你人工审核并微调，生成一个精美的 PDF 或 GPX 文件发给用户。
    4.  **收费点**：进群费 ¥9.9 或 定制路书 ¥19.9/次。
*   **目的**：**验证需求**。如果不付费，说明需求是伪命题，我们也没亏代码钱。

### 阶段二：静态网站 + 社区投稿 (3-6 个月)
*   **产品形态**：基于 Vue3 + Vercel 的响应式网站。
*   **核心功能**：
    *   **路线展示**：只展示最经典的 50 条“官方认证”路线。
    *   **路况众包**：用户点击“报错”，通过简单的表单提交路况更新。
*   **获客方式**：
    *   将生成的路书制作成精美的长图/视频，发布到小红书/抖音，引流到网站或私域。

## 4. 关键动作 (Action Plan)

1.  **开发 "AI 游记清洗小工具" (本地运行脚本)**
    *   不用做网页版，写一个 Python 脚本给自己用。
    *   功能：把复制来的游记文本，自动提取坐标点，转成 GPX。
    *   **成本**：0元。

2.  **搭建 Notion/飞书 数据库**
    *   建立初始的“ADV 路线库”。包括：路线名、难度、关键地标、GPX下载链接。
    *   这可以直接作为 MVP 的前端展示给用户看。

3.  **Vercel + MapLibre 原型验证**
    *   用一天时间写一个 Demo，验证免费地图加载是否流畅，能否显示 GPX 轨迹。

## 5. 风险控制
*   **地图合规性**：在中国境内运营地图服务需要资质。
    *   *规避策略*：初期不以“地图导航App”自居，而是以“户外运动社区”或“路书分享平台”定位。**数据仅供参考，风险自负**。

**一句话总结**：
前 100 个用户不需要 App，只需要你在这个群里，用最快的速度把靠谱的 GPX 甩在他们脸上。
